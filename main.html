Help

▼ Show usage and phase reference

Phases: 0 = Other, 1 = Accumulation, 2 = Markup, 3 = Distribution, 4 = Markdown (5–7 reserved). Bear score 0–10000 (higher = more bearish). Risk level 0–10.

Submit: Enter phase, bear score, risk; optionally metadata hash (0x64 hex or any string, will be hashed) and ETH for fee. Batch submit allows multiple rows; fee is per assessment.

Dashboard: Aggregate score stats and latest oracle snapshot. Phase summary shows count/sum/min/max per phase; Phase thresholds show keeper-configured min/max bounds.

Lookup: Load a single assessment by ID. My assessments lists IDs from the connected wallet. Block range returns assessment IDs submitted in the given block range.

Config: Treasury, keeper, oracle, deploy block, fee, counter, treasury balance, paused flag. Connect wallet and set BearChecker contract address to use all features.

Snapshots: Oracle (or owner) can record cycle snapshots via the contract; they appear under Snapshots and in the Dashboard as latest snapshot.

Batch: Add rows for phase/score/risk; total ETH must cover (fee × number of rows). Max 8 rows per batch and 256 assessments per submitter in total.

market-raider — BearChecker UI. Submit market-cycle assessments (phase, bear score, risk level). Keeper sets phase thresholds; oracle can record snapshots.

Use Connect to link your wallet; enter the deployed BearChecker contract address above. All data is read from or written to the contract on the connected chain.
